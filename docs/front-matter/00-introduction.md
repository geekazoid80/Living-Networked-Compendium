# Introduction

## What This Is

The Living Networked Compendium is a community-built body of knowledge for network engineers, written to support learning from first principles through to professional practice. It covers data networking, carrier transport, network automation, security, and the applied skills required for specific engineering roles.

It is called "living" because it is never finished. The underlying technology evolves, errors get corrected, and new contributors add depth. Every module carries a version number and a `human_reviewed` flag so you know how much verification has gone into it.

## Who It Is For

The compendium is written for anyone who works with or wants to understand computer networks, from someone who has never configured a router to an experienced engineer who wants a structured reference. Modules are written in plain English and introduce technical terms when they are needed, not before. No prior networking knowledge is assumed at the start of the fundamentals track.

## How It Is Maintained

Content is drafted with AI assistance and reviewed by human contributors before being marked `human_reviewed: true`. Modules without that flag are drafts: factually checked against authoritative sources, but not yet reviewed by a subject-matter expert. Never rely on an unreviewed module for production decisions.

Licensing: module content is published under CC BY-SA 4.0. Code samples and scripts are under Apache 2.0. See each module's Attribution and Licensing section for source details.

## Three Kinds of Content

Each module contains up to three buckets of content that render differently depending on how you are reading:

**Bucket 1 - Core content:** The main explanation. Always visible, whether you are reading in a browser, a PDF, or slides.

**Bucket 2 - Supplementary notes:** Deeper detail, historical context, or edge cases. On the web these are collapsed by default and can be expanded. In this PDF they appear as indented notes below the relevant section. They are safe to skip on a first read.

**Bucket 3 - Cross-references:** Maintenance links between modules, used by contributors. These are web-only and stripped from the PDF entirely.

## Two Ways to Read

**Linear curriculum:** Follow a learning path from start to finish. Each module builds on the ones before it. Prerequisites are listed at the top of every module. The learning paths section (next) describes the available paths and their module sequences.

**Reference:** Jump directly to the topic you need using the table of contents. The module ID scheme (`NW-001`, `IP-001`, `CT-003`) is consistent across the web and PDF versions so you can cross-reference easily.
